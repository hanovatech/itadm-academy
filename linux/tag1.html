Was ist ein Betriebssystem?

Ein Betriebssystem (BS) ist die Vermittlerschicht zwischen Mensch und Hardware. Stell dir das BS wie den Hausmeister im Hochhaus vor: verteilt Strom, regelt wer welche Räume nutzt, sorgt für Ordnung. Ohne BS müsste jedes Programm direkt mit der Hardware reden — Chaos!

Anwendungen ← Word, Browser, Spotify ↑ API ← Schnittstelle "Wie rede ich mit dem BS?" ↑ Kernel ← Das Herz des Betriebssystems ↑ Hardware ← CPU, RAM, Festplatte

Die vier Hauptaufgaben eines BS

  • Prozessverwaltung — Wer darf gerade auf der CPU rechnen?
  • Speicherverwaltung — Wer kriegt wie viel RAM?
  • Geräteverwaltung — Welcher Drucker, welche USB-Maus?
  • Dateisystem — Wo liegen die Daten auf der Platte?
💡
Knaller-Fakt

Auf den Top500 Supercomputern der Welt läuft 100 % Linux. Auf 0 von 500 läuft Windows.

🎓
Trainer-Hinweis

Chat-Frage stellen: "Wo nutzt ihr schon Linux, ohne es zu wissen?" Antworten sammeln und auflösen: Android, Smart-TV, Router, Auto (Tesla, MBUX), Geldautomaten, ISS, jeder Webserver.

08:30 – 08:50

Linux-Geschichte als Story

1969 — In den Bell Labs (AT&T) bauen ein paar Entwickler ein neues Betriebssystem: Unix. Es ist gut. Es ist frei. Universitäten weltweit nutzen es zum Lernen.

1979 — AT&T merkt: "Hey, damit kann man Geld verdienen!" Unix wird kommerziell. Universitäten sind sauer.

1984 — Richard Stallman vom MIT sagt: "Wenn die nicht freie Software wollen, machen wir unsere eigene!" Er startet das GNU-Projekt und erfindet die GPL-Lizenz — die Idee, dass Software für immer frei bleiben muss.

Problem: GNU hat alle Werkzeuge fertig — aber kein Herzstück. Kein Kernel.

1991 — Ein finnischer Informatik-Student namens Linus Torvalds schreibt in seinem Studentenzimmer einen Kernel als Hobby. Postet im Internet:

📜
"Hello everybody, I'm doing a free operating system (just a hobby, won't be big and professional…)"

Heute, 35 Jahre später: Sein "Hobby" läuft auf 90 % des Internets.

Linux-Distributionen & warum Rocky?

Linux-Kernel (Linus Torvalds) │ ┌───────────────┼───────────────┐ │ │ │ Debian-Familie Red Hat-Familie Arch-Familie │ │ │ - Ubuntu - RHEL (€€€) - Arch - Mint - Fedora - Manjaro - Kali - CentOS (✝2021) - Rocky ✅ ← unser Kurs! - AlmaLinux

Warum Rocky Linux?

  1. Kostenlos, aber 1:1 kompatibel mit RHEL (Red Hat Enterprise Linux, ~800 €/Jahr)
  2. Im Enterprise-Markt riesig — etwa 35 % aller Linux-Server sind RHEL-Familie
  3. Im Bewerbungsgespräch zählt's — "Erfahrung mit Red Hat oder Rocky?" wird gefragt
🎯
Praxis-Tipp

Wer Rocky kann, kann automatisch auch RHEL und CentOS. Das ist Gold wert im Bewerbungsgespräch.

BegriffBedeutung

Guest / GastDie VM (Rocky Linux)
HypervisorDie Software, die VMs verwaltet (VMware Workstation)

11:45 – 12:00

Navigation mit cd & ls

Der Linux-Verzeichnisbaum

/ ← Wurzel (Root-Verzeichnis, nicht "root user"!) ├── bin/ ← Wichtige Programme ├── boot/ ← Kernel, Bootloader ├── etc/ ← Konfigurationsdateien (extremely tasty configs 😉) ├── home/ ← Home-Verzeichnisse aller User │ └── admin/ ← unser Home (= ~) ├── root/ ← Home von root ├── tmp/ ← Temporäre Dateien ├── usr/ ← User-Programme und Bibliotheken └── var/ ← Variable Daten (Logs, Datenbanken)
💡
Linux ist anders als Windows!
  • Trenner: / in Linux, \ in Windows
  • Linux ist case-sensitive: Datei.txt ≠ datei.txt
  • Es gibt nur einen Wurzelpfad / — keine Laufwerke wie C:\ oder D:\

Die wichtigsten Befehle

bash 
cd /etc          # ins /etc-Verzeichnis wechseln
ls               # was ist drin?
ls -l            # Long-Listing (Details!)
ls -la           # auch versteckte Dateien (.dotfiles)
ls -lh           # Human-readable (KB, MB)
ls -lah          # alles zusammen

cd ..            # ein Verzeichnis hoch
cd ~             # ins Home
cd               # auch ins Home (ohne Argument)
cd -             # zurück zum letzten Verzeichnis

Dateien und Ordner verwalten

Jetzt wird's praktisch. Wir lernen die essentiellen Datei-Befehle:

bash 
# Ordner erstellen
mkdir test1
mkdir -p projekte/2025/januar    # mit Unterordnern

# Leere Datei erstellen
touch datei1.txt

# Kopieren
cp datei1.txt kopie.txt
cp datei1.txt test1/             # in Ordner
cp -r ordner1 ordner2            # Ordner rekursiv kopieren

# Verschieben / Umbenennen
mv kopie.txt umbenannt.txt       # Umbenennen
mv umbenannt.txt test1/          # Verschieben

# Löschen
rm datei1.txt                    # Datei löschen
rm -i datei1.txt                 # mit Nachfrage
rmdir leerer_ordner              # NUR leere Ordner
rm -r ordner_mit_inhalt          # Ordner samt Inhalt
rm -rf ordner                    # FORCE - VORSICHT!
💀
Die gefährlichste Linux-Zeile

rm -rf / löscht euer GANZES Linux! Es gibt keinen Papierkorb in der Konsole. Was weg ist, ist weg. Tippt das niemals als root!

Der Editor nano

Wir brauchen einen Texteditor. Es gibt zwei berühmte: nano (einfach, freundlich) und vim (mächtig, aber Lernkurve wie Eiger-Nordwand). Heute: nano.

⚠️
Rocky Minimal hat kein nano

Auf Rocky Linux Minimal ist nano nicht vorinstalliert! Wir müssen es zuerst installieren:

sudo = Admin-Rechte · dnf install = Paket installieren · -y = automatisches Ja

echo & Wildcards

echo — der einfachste Befehl

bash 
echo "Hallo Welt"
echo "Heute ist $(date)"
echo "Mein User: $USER"

Wildcards (Platzhalter)

WildcardBedeutungBeispiel
* beliebig viele beliebige Zeichen ls *.txt — alle txt-Dateien
? GENAU EIN beliebiges Zeichen ls test?.txt — test1.txt bis test9.txt
[ ] ein Zeichen aus der Liste ls test[135].txt — nur 1, 3, 5

Live-Demo

bash 
cd /usr/bin
ls m*           # alles, was mit m beginnt
ls m?           # m + 1 beliebiges Zeichen
ls m??          # m + 2 beliebige Zeichen
ls [abc]*       # alles, was mit a, b oder c beginnt

cd /etc
ls *.conf       # alle .conf-Dateien
15:30 – 15:45

Ausgabeumleitung (Redirection)

Normalerweise gibt Linux alles auf den Bildschirm aus. Mit Redirection können wir diese Ausgabe in Dateien umlenken.

OperatorBedeutung
>stdout in Datei (überschreibt!)
>>stdout an Datei anhängen
2>stderr (Fehler) in Datei
<stdin aus Datei lesen
&>stdout + stderr zusammen

Live-Demo

bash 
# Ausgabe in Datei (überschreibt!)
ls -la > inhalt.txt
cat inhalt.txt

# Anhängen statt überschreiben
echo "Neuer Eintrag" >> inhalt.txt
cat inhalt.txt

# Fehler umleiten
cd /existiert_nicht 2> fehler.log
cat fehler.log

# Beides: stdout UND stderr
ls /etc /existiert_nicht &> alles.log
cat alles.log
💡
Profi-Tipp

> /dev/null bedeutet "wegwerfen". Manchmal will man Ausgabe gar nicht sehen — z.B. in Skripten, die im Hintergrund laufen.

Die wichtigsten Befehle des Tages zum Ausdrucken oder Speichern:

BefehlWas er macht
pwdAktuelles Verzeichnis anzeigen
whoamiEingeloggter User
hostnameName des Systems
dateDatum und Uhrzeit
cd PfadVerzeichnis wechseln
cd ..Ein Verzeichnis hoch
cd ~ oder cdIns Home
cd -Zum vorherigen Verzeichnis
lsVerzeichnisinhalt
ls -laDetailliert + versteckte Dateien
ls -lhHuman-readable Größen
mkdir NameVerzeichnis erstellen
mkdir -p a/b/cVerschachtelt erstellen
touch DateiLeere Datei erstellen
cp Q ZDatei kopieren
cp -r O1 O2Ordner rekursiv kopieren
mv Q ZVerschieben / Umbenennen
rm DateiDatei löschen
rmdir OrdnerLeeren Ordner löschen
rm -r OrdnerOrdner samt Inhalt löschen
cat DateiDateiinhalt anzeigen
nano DateiDatei editieren
echo "Text"Text ausgeben
echo "X" > DIn Datei schreiben (überschreibt)
echo "X" >> DAn Datei anhängen
Befehl 2> DFehler in Datei umleiten
ls *.txtWildcard *: alle .txt
ls test?.txtWildcard ?: ein Zeichen
ls test[135].txtWildcard []: Auswahl
sudo BefehlBefehl als Admin ausführen
sudo dnf install X -yPaket installieren
