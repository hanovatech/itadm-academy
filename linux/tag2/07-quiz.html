  1. Kapitel 01 Einfach

    1. Welche Wildcard steht für „genau ein beliebiges Zeichen"?

    ✓ Richtig! ? steht für genau ein Zeichen. * ist „beliebig viele Zeichen“, [ ] ist „eines von diesen Zeichen“, ~ ist das Home-Verzeichnis.
    ✗ Leider falsch. ? steht für genau ein Zeichen. * ist „beliebig viele Zeichen“, [ ] ist „eines von diesen Zeichen“, ~ ist das Home-Verzeichnis.

  2. Kapitel 01 Mittel

    2. Du willst die Ausgabe von ls in eine Datei liste.txt umleiten und Fehlermeldungen verwerfen. Welcher Operator gehört in die Lücke?

    ls /etc > liste.txt ___ /dev/null

    ✓ Richtig! 2> leitet nur Fehler (stderr) um. > würde die normale Ausgabe nochmal überschreiben, >> würde sie anhängen, und &> würde alles zusammen umleiten.
    ✗ Leider falsch. 2> leitet nur Fehler (stderr) um. > würde die normale Ausgabe nochmal überschreiben, >> würde sie anhängen, und &> würde alles zusammen umleiten.
    Nochmal nachschauen: → Kapitel 01 (Wildcards, Redirection & Pipes)

    3. Was bedeutet die Option -z in tar?

    ✓ Richtig! -z aktiviert die gzip-Komprimierung. „Z“ wie „Zip“.
    ✗ Leider falsch. -z aktiviert die gzip-Komprimierung. „Z“ wie „Zip“.

  4. Kapitel 02 Mittel

    4. Welcher Befehl entpackt das Archiv backup.tar.gz?

    ✓ Richtig! -x = extract (entpacken). -c wäre erstellen, -t nur listen. gunzip kann nur einzelne .gz-Dateien, aber kein .tar-Archiv darin.
    ✗ Leider falsch. -x = extract (entpacken). -c wäre erstellen, -t nur listen. gunzip kann nur einzelne .gz-Dateien, aber kein .tar-Archiv darin.
    Nochmal nachschauen: → Kapitel 02 (Archivieren mit tar & gzip)
  5. Kapitel 03 Einfach

    5. Wahr oder Falsch: „Ein Hard-Link auf eine Datei und das Original haben dieselbe Inode-Nummer."

    ✓ Richtig! Ein Hard-Link ist ein zweiter Name für dieselbe Inode — keine Kopie, sondern eine zusätzliche Referenz. Beide zeigen auf exakt dieselben Datenblöcke.
    ✗ Leider falsch. Ein Hard-Link ist ein zweiter Name für dieselbe Inode — keine Kopie, sondern eine zusätzliche Referenz. Beide zeigen auf exakt dieselben Datenblöcke.
    Nochmal nachschauen: → Kapitel 03 (Inodes & Links)
  6. Kapitel 03 Mittel

    6. Du hast einen Soft-Link current → version-2.3.txt. Was passiert, wenn du version-2.3.txt löschst?

    ✓ Richtig! Soft-Links sind nur Wegweiser zu einem Pfad — wenn das Ziel weg ist, zeigt der Wegweiser ins Leere. Das nennt man „dangling symlink“. cat current würde dann fehlschlagen.
    ✗ Leider falsch. Soft-Links sind nur Wegweiser zu einem Pfad — wenn das Ziel weg ist, zeigt der Wegweiser ins Leere. Das nennt man „dangling symlink“. cat current würde dann fehlschlagen.
    Nochmal nachschauen: → Kapitel 03 (Inodes & Links)
  7. Kapitel 04 Einfach

    7. Du siehst in ls -l: drwxr-xr-x. Was für ein Objekt ist das?

    ✓ Richtig! Das erste Zeichen d steht für directory (Verzeichnis). - wäre eine normale Datei, l ein Symlink.
    ✗ Leider falsch. Das erste Zeichen d steht für directory (Verzeichnis). - wäre eine normale Datei, l ein Symlink.
    Nochmal nachschauen: → Kapitel 04 (Berechtigungen lesen)
  8. Kapitel 04 Mittel

    8. Welche Berechtigung steht hinter -rw-r-----?

    ✓ Richtig! Aufgeteilt in 3+3+3: rw- (User), r-- (Group), --- (Others). Dateityp - (normale Datei) wird abgetrennt vom ersten Zeichen.
    ✗ Leider falsch. Aufgeteilt in 3+3+3: rw- (User), r-- (Group), --- (Others). Dateityp - (normale Datei) wird abgetrennt vom ersten Zeichen.
    Nochmal nachschauen: → Kapitel 04 (Berechtigungen lesen)
  9. Kapitel 05 Mittel

    9. Welche Zahl entspricht der Berechtigung rwxr-x---?

    ✓ Richtig! Owner = rwx = 7, Group = r-x = 5, Others = --- = 0 → zusammen 750.
    ✗ Leider falsch. Owner = rwx = 7, Group = r-x = 5, Others = --- = 0 → zusammen 750.
    Nochmal nachschauen: → Kapitel 05 (Berechtigungen setzen)
  10. Kapitel 05 Schwer

    10. Du willst eine Datei privat machen — nur du darfst lesen und schreiben, sonst niemand. Welcher Befehl ist falsch? (Achtung: hier suchst du die falsche Antwort!)

    ✓ Richtig! 700 setzt auch x (Ausführrecht) für Owner — bei einer normalen Textdatei brauchst du das nicht. Korrekt wäre 600. Die anderen drei Varianten ergeben alle -rw-------.
    ✗ Leider falsch. 700 setzt auch x (Ausführrecht) für Owner — bei einer normalen Textdatei brauchst du das nicht. Korrekt wäre 600. Die anderen drei Varianten ergeben alle -rw-------.
    Nochmal nachschauen: → Kapitel 05 (Berechtigungen setzen)
  11. Kapitel 06 Einfach

    11. Wahr oder Falsch: „locate durchsucht das Dateisystem immer live und ist daher schneller als find."

    ✓ Richtig! locate ist schneller, aber nicht weil es live sucht — im Gegenteil: locate nutzt eine vorgenerierte Datenbank (täglich aktualisiert per Cron). Deswegen findet es eben keine neuen Dateien, die seit dem letzten updatedb erstellt wurden. find ist langsam, weil es das Dateisystem live durchsucht.
    ✗ Leider falsch. locate ist schneller, aber nicht weil es live sucht — im Gegenteil: locate nutzt eine vorgenerierte Datenbank (täglich aktualisiert per Cron). Deswegen findet es eben keine neuen Dateien, die seit dem letzten updatedb erstellt wurden. find ist langsam, weil es das Dateisystem live durchsucht.
    Nochmal nachschauen: → Kapitel 06 (Dateien suchen)
  12. Kapitel 06 Mittel

    12. Welcher Befehl findet alle .conf-Dateien in /etc/, die in den letzten 7 Tagen geändert wurden?

    ✓ Richtig! -mtime -7 heißt „weniger als 7 Tage her“ — also in den letzten 7 Tagen. +7 wäre „älter als 7 Tage“. locate hat keine -mtime-Option, das geht nur mit find.
    ✗ Leider falsch. -mtime -7 heißt „weniger als 7 Tage her“ — also in den letzten 7 Tagen. +7 wäre „älter als 7 Tage“. locate hat keine -mtime-Option, das geht nur mit find.
    Nochmal nachschauen: → Kapitel 06 (Dateien suchen)
